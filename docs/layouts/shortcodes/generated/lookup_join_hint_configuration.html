Key Default Type Description
async
 (none) Boolean Value can be 'true' or 'false' to suggest the planner choose the corresponding lookup function. If the backend lookup source does not support the suggested lookup mode, it will take no effect.
capacity
 (none) Integer The max number of async i/o operation that the async lookup join can trigger.
fixed-delay
 (none) Duration Delay time for the 'fixed-delay' retry strategy.
lookup-table
 (none) String The table name of the lookup source.
max-attempts
 (none) Integer Max attempt number of the 'fixed-delay' retry strategy.
output-mode
 (none)

Enum

 Output mode for asynchronous operations which will convert to {@see AsyncDataStream.OutputMode}, ORDERED by default. If set to ALLOW_UNORDERED, will attempt to use {@see AsyncDataStream.OutputMode.UNORDERED} when it does not affect the correctness of the result, otherwise ORDERED will be still used.

Possible values:
  • "ORDERED"
  • "ALLOW_UNORDERED"
retry-predicate
 (none) String A predicate expresses the retry condition, can be 'lookup-miss' which will enable retry if lookup result is empty.
retry-strategy
 (none)

Enum

 The retry strategy name, can be 'fixed-delay' for now.

Possible values:
  • "FIXED_DELAY"
timeout
 (none) Duration Timeout from first invoke to final completion of asynchronous operation, may include multiple retries, and will be reset in case of failover.
